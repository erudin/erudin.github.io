Imperative knowledge

Es una receta o "how-to".

Ejemplo numérico

Encontrar la raiz cuadrada de un número x es y tal que y*y = x

Receta para deducir la raiz cuadrada de un número x p.e. 16

  1. Empezar con un guess, g.
  2. Si g*g es lo suficientemente aproximado a x, detenerse y decir que ges la respuesta.
  3. De otra forma crear un nuevo guess promediando g y x/g.
  4. Utilizar el nuevo guess, y repetir el proceso hasta que sea lo suficientemente aproximado.
g g*g x/g (g+x/g)/2
3 9 5.333 4.1667
4.1667 17.36 3.837 4.0035
4.0035 16.0277 3.997 4.000002

¿Qué es una receta?

  1. Es una secuencia de simples pasos.
  2. Un proceso de flujo de control que especifica cuando se ejecuta cada paso.
  3. Un medio para determinar cuando detenerse.

Pasos 1+2+3 = un algoritmo!

Programas en Python

  • Un programa es una secuencia de definiciones y comandos.
    • Las definiciones son evaluadas.
    • Los comandos son ejecutados por el interprete de Python en una shell.
  • Los comandos le dicen al intérprete que haga algo.
  • Pueden ser escritos directamente en una shell o guardados en un archivo que es leído dentro de la shell y evaluado.

Objetos

  • Los programas manipulan objetos de datos.
  • Los objetos tienen un tipo que define el tipo de cosas que el programa puede hacer con ellos.
  • Los objetos son:
    • Escalares (no pueden ser subdivididos).
    • No escalares (tienen una estructura interna, a la cual se puede acceder.

Objetos escalares

  • int: representa los enteros, p.e. 5
  • float: representa los números reales, p.e. 5.5
  • bool: representa valores booleanos, True y False
  • NoneType: tipo especial y tiene un solo valor, None
  • Se puede utilizar type() para ver el tipo de un objeto

Conversión de tipos de objeto (cast)

Se puede convertir el tipo de un objeto a otro

Imprimiendo en la consola

Para mostrar el output del código a un usuario, se utiliza el comando print

Expresiones

  • Se pueden combinar objetos y operadores para formar una expresión.
  • Una expresión tiene un valor, el cual tiene un tipo.
  • La sintaxis para una simple expresión es la siguiente: < object > < operator > < object >

Operadores en los int y float

  • i+j -> suma.
  • i-j -> resta.
  • i*j -> multiplicación.
  • i/j -> división (float).
  • i//j -> división entera (int).
  • i%j -> residuo.
  • i**j -> potencia.

Simples operaciones

  • Los paréntesis suelen indicarle a Python cual operación realizar primero.
    • 3*5+1 es igual a 16
    • 3*(5+1) es igual a 18
  • Precedencia de los operadores sin paréntesis:
    • **
    • * y / -> se ejecutan de izquierda a derecha, como aparece en la expresión
    • + y -

Abstrayendo expresiones

¿Por qué darle nombres a los valores de las expresiones?

  • Se pueden reutilizar los nombres en vez de los valores.
  • Mas fácil para hacer debug y cambiar el código más adelante.

                            pi = 3.14159
                            radius = 2.2

Changing bindings

Se puede hacer un re-bind de el nombre de la variable usando una nueva asignación.

  • El valor anterior seguirá alojado en memoria pero no podrá accederse a el.
  • El valor del área (ejemplo) no cambiará hasta no decirle a la computadora que vuelva a realizar nuevamente el cálculo.

                            pi = 3.14
                            radius = 2.2
                            area = pi*(radius**2)

Operadores de comparación

  • i y j pueden ser cualquier nombre de variables.
    • i > j
    • i >= j
    • i < j
    • i <= j
    • i == j -> test de igualdad, True si i es igual a j.
    • i != j -> test de desigualdad, True si i no es igual a j.

Operadores lógicos en booleanos

  • a y b pueden ser cualquier nombre de variables.
    • not a -> True si a es False, False si a es False
    • a and b -> True si ambos son True
    • a or b -> True si uno de ellos o ambos son True

Branching Programs

  • La declaración de branching más simple es un condicional.
    • Un test (expresión que evalua a True o False).
    • Un bloque de código que se ejecuta si el test es True.
    • Un bloque de código adicional que se ejecuta si el test es False.

Ejemplo sencillo


                            x = int(input('Ingrese un entero: '))
                            if x % 2 == 0:
                                print('Es par')
                            else:
                                print('Es impar')
                            print('Listo con los condicionales')

Condicionales anidados


                            if x % 2 == 0:
                                if x % 3 == 0:
                                    print('Divisible entre 2 y 3’)
                                else:
                                    print('Divisible entre 2 y no entre 3’)
                            elif x % 3 == 0:
                                print('Divisible entre 3 y no entre 2’)

Semántica estática

Significa que una cadena de caracteres que es sintácticamente válida tiene significado.

Español:
"Yo quieres programar" -> válido sintácticamente pero con error de semántica estática.
Lenguaje de programación:
3+5 -> válido sintácticamente.
3+"hi" -> error de semántica estática.

Semántica

Es el significado asociado a una cadena de símbolos correcta sintácticamente, sin error de semántica estática.

Español:
"Volaré al subte a ver si lo logro tomar" -> puede tener diferentes significados.
Lenguaje de programación:
Solo tiene un significado pero puede que no sea lo que pretendia la persona que programa.

Resumen

  • Errores sintácticos: Comunes y fáciles de encontrar.
  • Errores de semántica estática:
    • Algunos lenguajes suelen revisarlos antes de correr el programa.
    • pueden ocasionar un comportamiento impredecible.
  • Sin error de semántica pero con diferente significado a lo que se pretendió programar:
    • El programa crashea, o detiene el proceso.
    • El programa corre infinitamente.
    • El programa arroja una respuesta pero diferente a la esperada.

Flujo de control

while loops


                            while < condition >:
                                < expression >
                                < expression >
                                ...
  • < condition > evalúa a boolean.
  • Si < condition > es True, se ejecutan todos los pasos del bloque de código dentro del while.
  • Evalúa nuevamente < condition >.
  • Se repite hasta que < condition > es False.

Ejemplo

Estás perdido en un bosque y si vas a la derecha seguirás dentro del bosque.


                            n = input('Estás perdido en el bosque. Ir a la izquierda o a la derecha? ')
                            while n == 'derecha':
                                n = input('Estás perdido en el bosque. Ir a la izquierda o a la derecha? ')
                            print('Has salido del bosque')

Flujo de control

for loops


                            for < variable > in range(< some_num >):
                                < expression >
                                < expression >
                                ...
  • < variable > toma un valor, cada vez que se ejecuta el loop.
  • La primera vez, < variable > empieza con el valor más pequeño.
  • La próxima vez, < variable > tomará el valor previo + 1.

range(start,stop,step)

  • Los valores por defecto son: start = 0 y step = 1(opcional)
  • El loop se ejecuta hasta que el valor es stop = -1

                            mysum = 0
                            for i in range(7, 10):
                                mysum += i
                            print(mysum)

                            mysum = 0
                            for i in range(5, 11, 2):
                                mysum += i
                            print(mysum)

break STATEMENT

  • Inmediatamente sale del loop.
  • Salta todos los bloques de código restantes.
  • Sale solamente del loop más interno.

for VS while LOOPS

  • for loop:
    • Saber el número de iteraciones.
    • Puede detenerse antes con un break.
    • Utiliza un contador.
    • Se puede reescribir un for loop utilizando un while loop.
  • while loop:
    • Número ilimitado de iteraciones
    • Puede detenerse antes con un break.
    • Puede utilizar un contador, pero debe ser inicializado antes del loop y ser incrementado dentro del loop.
    • Posiblemente no se pueda reescribir un while loop con un for loop.

Iteraciones

  • Comienza con un test.
  • Si evalúa a True, se ejecuta el cuerpo del loop una vez y vuelve a evaluar el test.
  • Esto se repite hasta que el test evalúa a False, y el código que sigue a la iteración es ejecutado.

Ejemplo


                            x = 3
                            ans = 0
                            itersLeft = x
                            while (itersLeft != 0):
                                ans = ans + x
                                itersLeft = itersLeft – 1
                            print(str(x) + '*' + str(x) + ' = ' + str(ans))

Éste código evalúa x al cuadrado a través de suma repetitiva.

Resumen

Algunas propiedades de las iteraciones:

  • Se necesita declarar una variable para iterar fuera del loop.
  • Se necesita probar la variable para determinar cuando termina.
  • Se necesita cambiar la variable en el loop

Las iteraciones nos permiten repetir pedazos de código hasta que una condición es cumplida.

Guess and Check

Encontrando la raiz cúbica de un entero

  • Una forma de utilizar la idea de generar guesses para encontrar la raiz cúbica de x es primero probar 0**3, luego 1**3, luego 2**3, y así sucesivamente.
  • Detenerse cuando encuentra a k, tal que k**3 > x

Ejemplo


                            x = int(input('Enter an integer: '))
                            ans = 0
                                while ans**3 < x:
                                    ans = ans + 1
                                if ans**3 != x:
                                    print(str(x) + ' is not a perfect cube')
                                else:
                                    print('Cube root of ' + str(x) + ' is ' + str(ans))

Solo funcionaría para enteros positivos.


                            x = int(input('Enter an integer: '))
                            ans = 0
                            while ans**3 < abs(x):
                                ans = ans + 1
                            if ans**3 != abs(x):
                                print(str(x) + ' is not a perfect cube')
                            else:
                                if x < 0:
                                    ans = - ans
                                print('Cube root of ' + str(x) + ' is ' + str(ans))

Resumen

  • Se puede "adivinar" el valor de una solución.
  • Se puede revisar si la solución es la correcta.
  • Se puede continuar suponiendo hasta encontrar la solución o "adivinar" todos los valores.
  • El proceso requiere enumeración exhaustiva.

Guess and Check con for loop

Una forma mucho más prolija


                            cube = 8
                            for guess in range(cube+1):
                                if guess**3 == cube:
                                    print("Cube root of ", cube, ” is ", guess)


                            cube = 8
                            for guess in range(abs(cube)+1):
                                if guess**3 >= abs(cube):
                                    break
                            if guess**3 != abs(cube):
                                print(cube, ’is not a perfect cube’)
                            else:
                                if cube < 0:
                                    guess = -guess
                                print('Cube root of ' + str(cube) + ' is ' + str(guess))
